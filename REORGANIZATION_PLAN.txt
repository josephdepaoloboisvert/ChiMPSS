1.  CURRENT STATE ASSESSMENT
================================================================================

Problems ranked by severity:

HIGH
  - No __init__.py files anywhere. Package imports rely on fragile
    sys.path.append() hacks and try/except fallback patterns.
  - Six independent copies of printf() defined as lambdas.
  - get_positions_from_pdb(), restrain_atoms(), unpack_infiles() are
    duplicated verbatim between MotorRow_utils.py and Minimizer_utils.py.
  - write_FASTA() is duplicated in utils/utils.py and utility/FileManipulations.py.
  - Two directories serve overlapping purposes: utils/ and utility/.
  - Module-level executable code in FultonMarketAnalysis.py (JAX init),
    utils/utils.py (rdkit config), Ligand/Ligand.py (rdkit config),
    calc_distance_matrix.py, grid_potentials.py, recovery.py,
    test_retro_conv.py.

MEDIUM
  - Mixed naming conventions: camelCase functions (returnRdkitMol,
    getTorsionalDistanceMatrix, getAlphaCarbonDistanceMatrix),
    PascalCase filenames for utility modules (FileManipulations.py,
    DistanceMatrixUtils.py), snake_case elsewhere.
  - MotorRow.py and MotorRow_utils.py sit at the repo root instead
    of in their own package directory.
  - 29 Jupyter notebooks scattered across root and subdirectories,
    many are experiments, not maintained examples.
  - GetContactsHelper.py at root duplicates/predates
    FultonMarket/ContactNetwork.py.
  - Two copies of ContactNetworkBuilder class (GetContactsHelper.py
    at root and ContactNetwork.py in FultonMarket/).
  - @staticmethod decorators on module-level functions in FultonMarketUtils.py.
  - rmsd lambda defined then overridden by a def in FultonMarketUtils.py.
  - build_thermodynamic_states() in FultonMarketUtils.py takes self but
    is a module-level function — dead code.

LOW
  - Inconsistent docstring depth (Bridgeport thorough, Minimizer absent).
  - .ipynb_checkpoints committed in multiple directories.
  - Test_Things_Work.py marked WIP throughout — stale.
  - interactive_utils.py read_json has a hardcoded filename bug.

4.  PHASE 2 — PACKAGE STRUCTURE
================================================================================

Goal: Make ChiMPSS a proper Python package with __init__.py files and
clean imports. Remove all sys.path.append() hacks.

4.1  Create __init__.py for every package directory.
  Each __init__.py should export the primary public class(es):

    chimpss/__init__.py:
      from .bridgeport import Bridgeport
      from .motor_row import MotorRow
      from .fulton_market import FultonMarket, FultonMarketAnalysis

    chimpss/bridgeport/__init__.py:
      from .bridgeport import Bridgeport

    chimpss/motor_row/__init__.py:
      from .motor_row import MotorRow

    chimpss/fulton_market/__init__.py:
      from .fulton_market import FultonMarket
      from .analysis import FultonMarketAnalysis

    chimpss/common/__init__.py:
      from .logging import printf

4.2  Replace every sys.path.append() with relative imports.
  Current offenders:
    - MotorRow.py:               sys.path.append(os.path.dirname(...))
    - Minimizer/Minimizer.py:    sys.path.append(split('/')[:-1])
    - utils/ProteinPreparer.py:  sys.path.append(split('/')[:-1])
    - Bridgeport/Bridgeport.py:  sys.path.append(split('/')[:-2])
    - Ligand/Ligand.py:          sys.path.append(split('/')[:-2]) AND [:-1]

4.3  Replace try/except import fallbacks with clean relative imports.
  Current offenders:
    - DistanceMatrix/DistanceMatrix.py:
        try: from .DistanceMatrixUtils import ...
        except: from DistanceMatrixUtils import ...

4.4  Replace `from X import *` with explicit imports.
  Current offenders:
    - MotorRow.py:               from MotorRow_utils import *
    - Minimizer.py:              from Minimizer_utils import *
    - FultonMarketwithAnalyzer:  from .FultonMarketUtils import *
    - Bridgeport.py:             from utils.utils import *
    - FileManipulations.py:      from .General import *

6.  PHASE 4 — MODULE-LEVEL SIDE EFFECTS REMOVAL
================================================================================

Goal: Importing a module should never trigger computation, GPU init,
filesystem writes, or global state mutation.

6.1  FultonMarketAnalysis.py — remove from module level:
    jax.print_environment_info()
    printf(f"Default JAX backend is {jax.default_backend()}")
  Move to FultonMarketAnalysis.__init__() or a dedicated setup function.

6.2  utils/utils.py and Ligand/Ligand.py — remove from module level:
    rdDepictor.SetPreferCoordGen(True)
    IPythonConsole.drawOptions.minFontSize=20
  These are display preferences; move to a notebook-side setup function or
  a lazy initializer that only runs when visualization is requested.

6.3  FultonMarketUtils.py — remove stale module-level objects:
    - @staticmethod decorators on module-level functions (remove decorators)
    - Dead lambda: rmsd = lambda ... (remove, keep only def rmsd)
    - Dead function: build_thermodynamic_states(self) (remove entirely)
    - perms = jnp.array([...]) at module level triggers JAX array creation
      on import. Move into the function that uses it, or lazy-init.

6.4  Root-level scripts that execute on import:
    - calc_distance_matrix.py   -> wrap in if __name__ == '__main__':
    - grid_potentials.py        -> wrap in if __name__ == '__main__':
    - test_retro_conv.py        -> wrap in if __name__ == '__main__':
    - recovery.py               -> wrap in if __name__ == '__main__':

9.  PHASE 7 — ENTRY POINT SCRIPTS
================================================================================

All CLI scripts in scripts/ should follow a consistent pattern:

  - argparse for all parameters
  - from chimpss.X import Y (clean package import)
  - if __name__ == '__main__': guard
  - No module-level computation

The RUN_FULTONMARKET.py pattern (already updated) is the template.
Apply to all other entry points.

Consider adding a setup.py or pyproject.toml with console_scripts
entry points so users can install and run:
    chimpss-bridgeport input.json
    chimpss-motor-row system.pdb system.xml output/
    chimpss-fulton-market system.pdb system.xml output/

10.  MIGRATION NOTES & RISK ASSESSMENT
================================================================================

EXECUTION ORDER
  Phase 1 first (utilities) — everything else depends on it.
  Phase 2 second (__init__.py) — enables clean imports for all later phases.
  Phases 3-5 can proceed in parallel after 1 and 2.
  Phase 6 (relocation) should be last, after all internal references are
  updated, to avoid breaking imports mid-refactor.

RISKS
  - Bridgeport.py is very large. Splitting it is desirable but risky.
    Recommendation: do NOT split Bridgeport.py in this pass. Move it
    intact and fix its imports. Splitting is a future refactor.
  - Notebook migration will break anyone who has paths pointing to the
    old locations. Add a README note.
  - MotorRow and Minimizer share a lot of code. After consolidating
    their utils, consider whether Minimizer should simply be a method
    on MotorRow rather than its own class (it's just _minimize()).
  - The ForceFields/ XML data files (amber14/, lipid17/, tip3p.xml etc.)
    need to travel with the package. Ensure they're included in the
    package data configuration.

BACKWARD COMPATIBILITY
  - Keep a top-level shim for at least one release cycle:
      # ChiMPSS/FultonMarket/FultonMarketwithAnalyzer.py
      from chimpss.fulton_market.fulton_market import FultonMarket
    This lets existing notebooks and scripts continue to work while
    migrating to the new import paths.
  - RUN_FULTONMARKET_MPI.py calls RUN_FULTONMARKET.py by path via
    runpy. After relocation, update the path reference or convert
    to a package-level import.

TESTING
  - No test suite exists currently. Before major refactoring, create
    a minimal smoke test that imports each public class and verifies
    construction does not raise. This protects against import breakage.
  - After each phase, run the smoke test before proceeding.
